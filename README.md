# Project Wiki Bootstrap Skill

快速为一个新项目或旧代码仓库生成可维护的项目 Wiki：从技术栈、启动路径、核心模块、关键链路到风险和待确认项，帮助团队快速建立共同认知。

## 能做什么

- 快速梳理仓库结构和技术栈
- 输出从 clone 到运行的最短路径
- 建立模块地图、关键链路和排障入口
- 用“结论 / 证据 / 置信度”减少拍脑袋文档
- 生成适合团队共享的 `docs/PROJECT_WIKI.md`

## 使用方式

把本目录作为 skill 安装到支持 skills 的工具中，然后在项目根目录发起类似请求：

```text
使用 wiki skill，帮我为当前仓库生成一份 docs/PROJECT_WIKI.md，深度按 15 分钟速览。
```

也可以指定更深的目标：

```text
使用 wiki skill，做 1 小时深挖版 Wiki，重点梳理启动路径、核心链路和常见排障。
```

## 默认输出

- 项目概览
- 快速开始
- 仓库结构
- 架构与关键链路
- 开发指南
- 运维与排障
- 风险与待确认

## 设计原则

- 证据优先，不凭空猜测
- 可运行优先于漂亮描述
- 未验证内容明确标记为 `TODO/待确认`
- 不暴露密钥、客户数据或私有配置值
